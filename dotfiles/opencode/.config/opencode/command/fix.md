Fix the specified bug or error with comprehensive root cause analysis. Process:

1. **Reproduce**: Understand the bug, identify steps to reproduce, gather error messages/stack traces
2. **Root cause analysis**: Investigate why the bug occurs, trace through code execution, identify the underlying issue
3. **Fix implementation**: Provide a proper fix that addresses the root cause (not just symptoms)
4. **Prevention**: Suggest how to prevent similar bugs in the future (tests, type safety, validation)
5. **Testing**: Write tests that verify the fix and prevent regression
6. **Documentation**: Update docs if the bug revealed unclear behavior

Explain your reasoning clearly. Consider edge cases that might trigger similar issues. Avoid quick patches that don't address underlying problems.
